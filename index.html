What makes the Web Work?

The web is many computers that communicate with each other. When a person goes to a web page like www.yahoo.com, that persons computer sends a HTTP Request to a server.

The server will find the necessary HTML document and it sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen.

(HTML)

(HTML) stands for Hypertext Markup Language. HTML documents form the majority of the content on the web.

HTML documents contain text content which describes "what you see" and markup which describes "how it looks".

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.

Are Computers actually Stupid?

Computers are considered stupid because they interpret instructions. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

Inline and Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Developer Tools (in the Browser)

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree.

In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

What is the relationship between indented HTML and boxes??

When someone reads an HTML document as text, you will see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Text Editors that can be used...

When writing code, programmers can use special text editors (Sublime Text + many others). These editors make the programmer's life much easier. For example, some text editors will automatically generate a closing HTML tags (which can help drastically save time).

Avoid Repetition at all cost :-)

Avoiding repetition is important for a variety of reasons. Avoiding Errors: Most of the time programmers don't avoid repetition, a lot of times they have do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) over and over again. This can open up the door for more errors when a programmer decides to change something. If they don't dilligently make that same change everywhere the repeated code appears, problems will crop up.
